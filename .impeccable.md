## Design Context

### Users
Stow is for people who need household inventory to be fast enough to use in real life: renters, homeowners, families, frequent movers, travelers, and anyone trying to remember what they own and where it lives. They use it while standing in closets, unpacking boxes, making packing lists, documenting important objects, or searching later under mild stress.

### Brand Personality
Calm, exact, and quietly warm. Stow should feel like a well-run studio assistant: precise about location and evidence, forgiving about incomplete capture, and confident without sounding clever.

### Aesthetic Direction
The next redesign should feel like a refined command desk for physical belongings: light, tactile, information-dense, and composed. It should avoid generic dashboard cards, purple-blue gradients, cute narration, and marketing hero layouts. The interface should make capture feel immediate, organization feel spatial, and evidence/status feel trustworthy but unobtrusive.

### Design Principles
- Capture stays first, but the user always remains in control before anything is saved.
- Location is a first-class object: spaces, areas, drafts, and items should be visually connected.
- Forms stay short by default with persistent labels, optional details tucked into secondary controls.
- Desktop should use multiple visible work regions; mobile should keep the same jobs but use bottom navigation and thumb-safe actions.
- Evidence is quiet, not punitive: skipped/later photos and AI boundaries should read as status, not failure.
